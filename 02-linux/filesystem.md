# Linux Filesystem

## The Filesystem Hierarchy Standard (FHS)

Linux organizes all files in a single unified directory tree rooted at `/`. Physical disks, network shares, and virtual filesystems are all mounted at points within this tree.

```
/
├── bin/        Essential user commands (ls, cp, mv, grep)
├── sbin/       System administration commands (mount, fdisk, ifconfig)
├── etc/        System-wide configuration files
├── home/       User home directories (/home/ubuntu, /home/ec2-user)
├── root/       Root user's home directory
├── var/        Variable data — logs, caches, spool files
├── tmp/        Temporary files (cleared on reboot on most distros)
├── run/        Runtime data — PIDs, sockets (tmpfs, cleared on reboot)
├── usr/        Secondary hierarchy — most installed software
│   ├── bin/    Non-essential user commands
│   ├── sbin/   Non-essential system commands
│   └── lib/    Libraries for /usr/bin and /usr/sbin
├── lib/        Shared libraries for /bin and /sbin
├── dev/        Device files (block devices, character devices)
├── proc/       Virtual filesystem — process and kernel information
├── sys/        Virtual filesystem — hardware and kernel subsystem info
├── opt/        Optional third-party software (rare in modern distros)
├── mnt/        Temporary mount points
├── media/      Removable media mount points (USB, CD-ROM)
└── boot/       Bootloader and kernel files
```

## Important Directories for Cloud Engineers

### `/etc` — Configuration

All system-wide configuration lives here. Editing files in `/etc` changes system behaviour.

```bash
/etc/passwd          # User account info (username, UID, home, shell)
/etc/shadow          # Hashed passwords (readable only by root)
/etc/group           # Group definitions
/etc/sudoers         # Who can use sudo and with what restrictions
/etc/ssh/sshd_config # SSH server configuration
/etc/hostname        # System hostname
/etc/hosts           # Static DNS mappings
/etc/resolv.conf     # DNS resolver configuration
/etc/fstab           # Filesystem mount table (persists across reboots)
/etc/crontab         # System-wide cron jobs
/etc/environment     # System-wide environment variables
/etc/os-release      # Distro name and version
```

### `/var` — Variable Data

Data that changes during normal operation.

```bash
/var/log/            # System and application logs
/var/log/syslog      # General system log (Ubuntu/Debian)
/var/log/messages    # General system log (RHEL/Amazon Linux)
/var/log/auth.log    # Authentication events (sudo, SSH logins)
/var/log/secure      # Auth events on RHEL/CentOS
/var/log/nginx/      # Nginx access and error logs
/var/log/cloud-init.log  # Cloud-init output (user data script execution)
/var/lib/            # Persistent application state (databases, package DBs)
/var/cache/          # Cached data (apt cache, yum cache)
/var/spool/          # Queued data (cron jobs, print spooler, mail)
/var/run/            # Runtime PID files and sockets (symlink to /run)
```

### `/proc` — Process and Kernel Information (Virtual)

`/proc` is not on disk — it's generated by the kernel in memory. Used for querying system state without special tools.

```bash
/proc/cpuinfo        # CPU model, cores, flags
/proc/meminfo        # Memory usage breakdown
/proc/net/           # Network interface and connection tables
/proc/sys/           # Tunable kernel parameters (sysctl)
/proc/{PID}/         # Per-process info (maps, fd, status, environ)
/proc/{PID}/cmdline  # Command line of a running process
/proc/{PID}/environ  # Environment variables of a running process
/proc/loadavg        # System load averages (1m, 5m, 15m)
/proc/uptime         # System uptime in seconds
```

```bash
# Practical examples
cat /proc/cpuinfo | grep "model name" | head -1
cat /proc/meminfo | grep MemAvailable
cat /proc/loadavg
cat /proc/$(pgrep nginx)/cmdline | tr '\0' ' '
```

## Navigating the Filesystem

```bash
# Current location
pwd

# List files
ls                   # simple list
ls -l                # long format (permissions, owner, size, date)
ls -la               # include hidden files (dot files)
ls -lh               # human-readable sizes (KB, MB, GB)
ls -lt               # sort by modification time (newest first)
ls -lS               # sort by size (largest first)

# Change directory
cd /var/log          # absolute path
cd ../..             # two levels up
cd -                 # return to previous directory
cd ~                 # go to home directory

# Find files
find /var/log -name "*.log" -mtime -1    # logs modified in last 1 day
find /home -name ".ssh" -type d          # find .ssh directories
find / -perm /4000 -type f               # find SUID files
find . -size +100M                       # files larger than 100MB
find /tmp -mtime +7 -delete              # delete tmp files older than 7 days

# Locate files (uses database, faster than find)
locate nginx.conf
sudo updatedb                            # update locate database
```

## Viewing and Manipulating Files

```bash
# View file contents
cat /etc/os-release
less /var/log/syslog          # paginated view (q to quit, / to search)
head -20 /var/log/auth.log    # first 20 lines
tail -50 /var/log/nginx/error.log        # last 50 lines
tail -f /var/log/nginx/access.log        # follow (live tail — Ctrl+C to stop)
tail -f /var/log/syslog | grep ERROR     # live tail filtered by pattern

# Search in files
grep "error" /var/log/syslog
grep -i "error" /var/log/syslog          # case-insensitive
grep -r "DB_PASSWORD" /etc/              # recursive search
grep -n "failed" /var/log/auth.log       # show line numbers
grep -v "healthcheck" /var/log/nginx/access.log  # exclude pattern
grep -c "ERROR" /var/log/app.log         # count matches

# Text processing
awk '{print $1}' /var/log/nginx/access.log   # print first field (IP)
awk -F: '{print $1}' /etc/passwd             # print usernames (colon delimited)
sed 's/old/new/g' config.txt                  # replace text
sed -i 's/DEBUG/INFO/g' app.conf              # in-place replace
cut -d: -f1,3 /etc/passwd                    # cut columns 1 and 3
sort /etc/hosts | uniq                        # sort and deduplicate
wc -l /var/log/auth.log                       # count lines

# Copy, move, delete
cp file.txt /tmp/
cp -r /etc/nginx /tmp/nginx-backup            # recursive copy
mv old-name.txt new-name.txt
rm file.txt
rm -rf /tmp/old-directory                     # force recursive delete (CAREFUL)
```

## Disk Usage

```bash
# Disk free — how much space is available
df -h                            # all mounted filesystems, human-readable
df -h /                          # root filesystem only
df -i                            # inode usage (relevant if you have many small files)

# Disk usage — what's consuming space
du -sh /var/log                  # total size of /var/log
du -sh /var/log/*                # size of each item inside
du -h --max-depth=1 /            # size of each top-level directory
du -sh * | sort -rh | head -20   # top 20 largest items in current dir

# Find large files quickly
find / -type f -size +500M -exec ls -lh {} \; 2>/dev/null

# Check inode usage (when disk shows full but df -h shows space)
df -i
find / -xdev -printf '%h\n' | sort | uniq -c | sort -rn | head -20
```

## Mounting and Filesystems

### Common Filesystem Types

| Type | Description | Common use |
|------|-------------|-----------|
| `ext4` | Standard Linux filesystem | EC2 EBS volumes, root disk |
| `xfs` | High-performance, journaling | Amazon Linux default, large volumes |
| `tmpfs` | RAM-backed, volatile | `/tmp`, `/run` — fast but not persistent |
| `nfs` | Network File System | EFS mounts on EC2 |
| `overlay` | Layered filesystem | Docker container layers |
| `vfat` | FAT32 | `/boot/efi` partition |

### Mounting Filesystems

```bash
# Mount a device temporarily
sudo mount /dev/xvdb1 /mnt/data
sudo mount -t ext4 /dev/xvdb1 /mnt/data     # specify filesystem type

# Mount an NFS share (EFS)
sudo mount -t nfs4 \
  -o nfsvers=4.1,rsize=1048576,wsize=1048576,hard,timeo=600,retrans=2 \
  fs-abc123.efs.us-east-1.amazonaws.com:/ /mnt/efs

# Unmount
sudo umount /mnt/data
sudo umount -l /mnt/data                     # lazy unmount if busy

# View mounted filesystems
mount | column -t
cat /proc/mounts
```

### /etc/fstab — Persistent Mounts

Mounts defined in `/etc/fstab` are applied automatically at boot.

```
# Device              Mountpoint   Type    Options         Dump  Pass
/dev/xvda1            /            ext4    defaults        1     1
/dev/xvdb1            /data        xfs     defaults,nofail 0     2
tmpfs                 /tmp         tmpfs   defaults,size=2G 0    0
fs-abc123.efs.us-east-1.amazonaws.com:/ /mnt/efs nfs4 nfsvers=4.1,hard 0 0
```

> **`nofail` option**: Important for cloud VMs — if the EBS volume isn't attached, the system still boots. Without `nofail`, a missing mount in fstab can prevent boot.

```bash
# Test fstab without rebooting
sudo mount -a               # mount all entries in fstab
sudo mount -av              # verbose — shows what happened
```

## Links

### Hard Links

A hard link is an additional directory entry pointing to the same inode (same data on disk). Both names are equally valid.

```bash
ln original.txt hardlink.txt
ls -li original.txt hardlink.txt    # same inode number
rm original.txt                      # data still accessible via hardlink.txt
```

Hard links cannot span filesystems and cannot link directories.

### Symbolic (Soft) Links

A symlink is a file that contains a path to another file. More flexible but breaks if the target is moved or deleted.

```bash
ln -s /etc/nginx/nginx.conf nginx.conf   # create symlink
ls -la nginx.conf                         # shows: nginx.conf -> /etc/nginx/nginx.conf
readlink -f nginx.conf                    # resolve symlink to absolute path
```

Common in cloud: `/etc/nginx/sites-enabled/` contains symlinks to config files in `sites-available/`.

```bash
# System identification
cat /etc/os-release              # distro name and version
cat /etc/hostname                # hostname
```
